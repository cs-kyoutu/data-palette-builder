# 計算・フォーマット系パターン

## 売上/金額による優先判定パターン

「複数該当がある場合、金額が高いものを優先」のケース：

1. **ランキング**: グループ化キーで金額を大きい順にランキング（同率なし）
2. **絞込み**: ランキング = 1 で絞込み
3. **削除**: 不要なランキングカラム等を削除

---

## KPI計算の連鎖パターン（四則演算チェーン）

ROI等の複合KPIは四則演算を複数回チェーンして計算する：

1. 中間値A = カラム1 × カラム2
2. 中間値B = 中間値A ÷ カラム3
3. 最終値 = 中間値B × 100（パーセント化）

各ステップで端数処理（小数第N位、四捨五入/切り捨て）を指定。
表示方法「残す」で中間カラムを保持し、最後に不要カラムを削除。

---

## パーセント表示パターン

小数値（0.33等）をパーセント表示にする場合：
1. 四則演算: 値 × 100（小数第1位四捨五入）
2. テキスト挿入: 文末に「％」挿入
3. 元の小数カラムを削除

---

## 金額表示フォーマットパターン

金額カラムを表示用にフォーマットする場合：
1. 四則演算: 値 ÷ 1000（千円単位化、小数第1位切り捨て）
2. 型変換: 整数型に変換
3. テンプレート「金額をカンマ区切りにした値へ変換」

※テンプレートは整数型のみ適用可能。先に型変換が必要。

---

## 完全新規判定パターン

「初めての購入かどうか」を判定する場合：
1. ランキング: 顧客IDでグループ化、注文日の小さい順、同率なし
2. IF文: ランキング=1の場合に顧客IDを代入、それ以外は空白
3. 集約: IF文カラムをカウント → 完全新規数

---

## 受注回数の算出パターン（昇順・降順）

顧客ごとの受注回数（N回目の購入）を算出：
1. ランキング: 顧客IDでグループ化、受注日時の小さい順、同率なし → 受注回数（昇順）
2. ランキング: 顧客IDでグループ化、受注日時の大きい順、同率なし → 受注回数（降順）
3. IF文: 顧客IDが空文字の場合は"1"（ゲスト購入対応）
4. 型変換: 整数型に変換

新規/リピート判定: 受注回数=1なら"新規"、それ以外"リピート"

---

## 四則演算のチェーン加算（3つ以上の合計）

b→dashの四則演算は2カラム間の演算のみ。3つ以上のカラムを合計する場合：

1. A + B → パーツ1（表示方法: 残さない）
2. パーツ1 + C → パーツ2（表示方法: 残さない）
3. パーツ2 + D → パーツ3（表示方法: 残さない）
4. ...
5. パーツN-1 + 最後のカラム → 最終合計（表示方法: 残す）

中間パーツは「残さない」で元カラムを消しながらチェーン。最後だけ「残す」。

---

## 時刻演算のカスタム値パターン

「本日の日付」や「特定の日付」ではなく、固定の数値（月数、日数）で演算する場合：

```
『時刻演算』
「{column}」《-》《カスタム：""{value}""》　[計算結果]を《{unit}》で算出
```

例: 注文年月日 - カスタム:1 → 1ヶ月前の日付
    注文年月日 - カスタム:5 → 5ヶ月前の日付

---

## IF文の期間判定（絶対期間+カラム参照）

IF文で「本日の日付がカラムAとカラムBの期間内にあるか」を判定する場合：

```
『IF文』
《絶対期間》を選択
「本日の日付」が[開始日]「{start_column}」と[終了日]「{end_column}」《指定の期間に一致する》の場合、""{true_value}""に変換
いずれの条件分岐にも該当しない場合、""{false_value}""に変換
[開始日時/終了日時を含めない]にチェックを入れない
[期間指定にカラムを使用する]にチェックを入れる
```

開始/終了のどちらかがNULLの場合は個別にIF文を分ける必要がある：
- 開始あり・終了あり → 期間判定
- 開始あり・終了NULL → 開始日以降か判定
- 開始NULL・終了あり → 終了日以前か判定
- 両方NULL → product_statusで判定

---

## 連結をフラグ結合に使うパターン

2つの条件の AND判定 を「連結→特定値で絞込み」で実現：

1. IF文でフラグA作成（条件満たせば"1"、それ以外は空白）
2. IF文でフラグB作成（条件満たせば"1"、それ以外は空白）
3. 連結でフラグA+フラグBを結合（区切り文字なし）
4. 絞込みで"11"に完全一致するレコードのみ残す

"11" = 両方の条件を満たす。片方でも空白なら"1"や""になり除外される。

---

## IF文による値→カテゴリ変換パターン

市区町村等の細かい値を中カテゴリ（地域グループ等）に変換する場合：

```
『IF文』
「勤務地」が""値A,値B,値C""《次を含む(カンマ区切りで複数指定)》の場合、""カテゴリ1""に変換
「勤務地」が""値D,値E""《次を含む(カンマ区切りで複数指定)》の場合、""カテゴリ2""に変換
いずれの条件分岐にも該当しない場合、「勤務地」に変換
```

同じ変換ロジックを複数パーツで再利用する場合は、各パーツで同じIF文を記載。
「条件はID○○のタスクNo.○『IF文』を参考にする」と参照先を記載する場合もある。

---

## 年齢→年代変換のIF文パターン

年齢（整数）を年代（テキスト）に変換する場合、IF文で範囲条件を網羅：
- 0未満→"0未満"
- 0～9→"0~9歳"
- 10～19→"10代"
- 20～29→"20代"
- ...
- 100～109→"100歳~109歳"
- 110以上→"110歳以上"
- いずれにも該当しない→空白

---

## 開封者数/クリック者数のフラグ変換パターン

集約でカウント/ユニークカウントした後、0件でない場合に顧客IDを代入するパターン：

```
『IF文』
「開封数-計算用」が""0""《次と等しくない》の場合、「顧客ID」に変換
いずれの条件分岐にも該当しない場合、《空白》に変換
→「開封者数-計算用」
```

これで後続の集約で「開封者数-計算用」をユニークカウントすれば開封者数が出る。
