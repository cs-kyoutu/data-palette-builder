# テキスト操作・クリーンアップパターン

## 置換の正規表現パターン

置換で`《正規表現》`を使うとパターンマッチで値を変換できる：

```
『置換』
「{column}」の《正規表現》""{regex_pattern}""を""{replacement}""に置換
《上書き保存》にて保存
クレンジングタスクの保存名を""{task_name}""にする
```

用途: コード体系のグルーピング、マスキング、パターン変換等。
複数パターンがある場合は置換を繰り返し実行（上書き保存で1回ずつ）。

---

## 除外の「全ての」パターン

カラム内の特定文字列をすべて削除する場合：

```
『除外』
「{column}」の《全ての》""{text}""を除外
《上書き保存》にて保存
クレンジングタスクの保存名を""{task_name}""にする
```

例: 電話番号のハイフン除去（半角"-"と全角"ー"の2回実行）

---

## テキスト挿入でURL動的生成パターン

商品コードからURLを動的生成する場合：
- テキスト挿入: 文頭にベースURL、文末に拡張子を挿入
- 例: ITEM_CD → 文頭に`https://www.example.com/item/`、文末に`.html`
- 名前を付けて保存で新カラム「商品詳細URL」として作成

---

## 条件分岐+連結+テキスト挿入によるURL動的生成パターン

条件に応じてURLのベース部分を分岐し、IDを連結してURLを生成：

1. IF文: 条件に応じてベースURLを分岐（例: mail→/mails/campaign、scenario→/scenarios）
2. 連結: ベースURL + ID（区切り文字"/"）
3. テキスト挿入: 文末に"/detail"等のパスを追加

単純なテキスト挿入でのURL生成（ベースURLが固定）との違い：ベースURL自体が条件分岐する。

---

## URLエンコードの連続置換パターン

リレーション項目等にURLエンコードされた値が入っている場合、複数回の置換でデコード：
1. 置換: %2F → /
2. 置換: %3D → =
3. 置換: %2B → +
4. 置換: %20 → +（またはスペース）

各置換は上書き保存で個別実行。

---

## NULL/空文字の置換パターン

集計結果やLEFT JOINの結果でNULLが発生する場合：
- **置換**: NULLを特定値（"0"等）に変換
- **上書き保存**で元カラムに反映（過去1回以上クレンジング済みが条件）

---

## 型変換は1カラムずつ個別実行

型変換（特に上書き保存）は**1カラムにつき1回の加工保存**が必要。複数カラムの型変換をまとめて1回で実行できない。

例: 日付型に変換するカラムが5つある場合
1. 型変換: 最終対応日 → 日付型（上書き保存、タスク名「最終対応日」）
2. 型変換: 初回問合せ日 → 日付型（上書き保存、タスク名「初回問合せ日」）
3. 型変換: 最終問合せ日 → 日付型（上書き保存、タスク名「最終問合せ日」）
4. ...

型変換エラー処理は「Nullに変換する」を選択。

---

## NULL→0置換+型変換の大量繰り返しパターン

LEFT JOINで生じたNULLカラムが多数ある場合、各カラムに対して：
1. 置換: NULL→"0"（上書き保存）
2. 型変換: 整数型に変換（上書き保存）

カラム数×2回の加工が必要。手順書では全カラム分を連続記載する。

---

## 3カラム以上の連結

通常の連結は2カラムまで。3つ以上のカラムを連結する場合は専用テンプレートを使用：
- テンプレート「3つ以上のカラムやテキストを連結した値を追加」
- 区切り文字はテキスト挿入で指定

---

## 時刻帯での絞込みパターン（深夜帯・特定時間帯）

日時カラムの時刻部分だけで特定の時間帯に絞り込む場合：

### 手順
1. テキスト挿入: 日時カラムの文頭に固定日付"2000/01/01"を挿入（時刻部分だけ残す）
2. 型変換: 日時型に変換
3. IF文（絶対期間・固定値）: "2000/01/01 23:30:00"〜"2000/01/01 23:59:59"の期間判定→"1"
   - `[期間指定にカラムを使用する]にチェックを入れない`（固定値）
4. 必要なら複数の時間帯を別々にIF文（例: 00:00〜06:30も別途判定）

### IF文（絶対期間・固定値）の書き方
```
『IF文』
《絶対期間》を選択
「最新かご落ち日時_日時」が[開始日]""2000/01/01 23:30:00""と[終了日]""2000/01/01 23:59:59""《指定の期間に一致する》の場合、""1""に変換
いずれの条件分岐にも該当しない場合、《空白》に変換
[開始日時/終了日時を含めない]にチェックを入れない
[期間指定にカラムを使用する]にチェックを入れない
```

※カラム参照の期間判定とは異なり、固定値で日時を直接指定するパターン。
