# 誕生日施策パターン

誕生日N日前のセグメント条件を算出するための固定パターン。

---

## 「今年の誕生日」カラム作成手順（b→dashサポートサイト準拠）

以下の手順で「本年＋誕生日(日付型)」カラムを作成する。

### 1. 生年月日から月日を抽出
```
『抽出』
「生年月日」の《末尾》から""5""文字目までを《テキスト型》で抽出
《名前を付けて保存》にて保存
クレンジングタスクの保存名を""誕生月日""にする
```

### 2. うるう年対応
```
『IF文』
「誕生月日」が""02/29""《次に完全一致》の場合、""02/28""に変換
いずれの条件分岐にも該当しない場合、《元の値のまま》に変換
```
※2/29生まれは2/28として処理。カラム名は上書きまたは新カラム名を付ける。

### 3. 本日日付の追加と年の抽出
```
『追加』
「生年月日」の《右に追加》を選択
《日付型》を選択し、[加工処理実行日のカラムを追加する]にチェックを入れる
```
```
『抽出』
「追加列」の《先頭》から""5""文字目までを《テキスト型》で抽出
《名前を付けて保存》にて保存
クレンジングタスクの保存名を""本年""にする
```

### 4. 年と月日を連結
```
『連結』
「本年」と「誕生月日」を選択
クレンジングタスクの保存名を""今年の誕生日テキスト""にする
表示方法《残す》を選択
```

### 5. 日付型に変換
```
『型変換』
「今年の誕生日テキスト」を《日付型》に変換
型変換エラーの処理は、《クレンジングを中止する》を選択
《名前を付けて保存》にて保存
クレンジングタスクの保存名を""今年の誕生日""にする
```

### 6. 誕生日までの日数を算出
```
『時刻演算』
「今年の誕生日」《-》「本日日付」、[計算結果]を《日》で算出
クレンジングタスクの保存名を""誕生日までの日数""にする
表示方法《残す》を選択
```

### 7. 誕生月の抽出（必要な場合）
```
『抽出』
「誕生月日」の《先頭》から""2""文字目までを《テキスト型》で抽出
《名前を付けて保存》にて保存
クレンジングタスクの保存名を""誕生月""にする
```

---

## 年齢算出（テンプレート使用）

年齢が必要な場合はテンプレートを使う：
```
テンプレート『生年月日から年齢を算出』
「生年月日」が日付型カラムに選択され、「年齢」が追加されました
```

---

## セグメント条件の例

「誕生日が7日後の顧客」をセグメントにする場合：
- 「今年の誕生日」が《相対期間》[開始日]《7》《日》《後》と[終了日]《7》《日》《後》《次の期間にある》

「今月が誕生月の顧客」をセグメントにする場合：
- 「誕生月」が本月の値（例: ""03""）に完全一致

---

## 不要カラムの削除

最後に中間カラムを削除：
```
『削除』
「追加列」、「本年」、「誕生月日」、「今年の誕生日テキスト」を削除する
```
