# 横統合テンプレート

## フォーマット

```
『横統合』
【{left_file}】と【{right_file}】を「{key_left}」と「{key_right}」を統合キーとして、《{method}》
残すカラムは、
【{left_file}】：{left_columns}
【{right_file}】：{right_columns}
データファイルの保存名を""{save_as}""にする
レコードが重複した際の処理方法は
《{duplicate_handling}》を選択し、[適用]を押下する
統合データの更新設定は《{update_setting}》を選択し、[適用]を押下する
```

## 変数

| 変数 | 説明 | 例 |
|------|------|-----|
| left_file | 左（先に選択する）ファイル名 | 【順位3】空いたら登録者かつ対象物件統合データ |
| right_file | 右（後に選択する）ファイル名 | 物件マスタ |
| key_left | 左の統合キー | 空いたら物件ID |
| key_right | 右の統合キー | 物件ID |
| method | 統合方法 | 先に選択したデータに対して統合する / 共通のデータのみを統合する / 全てのデータを統合する / 後に選択したデータに対して統合する |
| left_columns | 左から残すカラム（「」区切り） | 「顧客ID」「メールアドレス」「姓」「名」 |
| right_columns | 右から残すカラム（「」区切り） | 「外観画像URL」「間取り画像URL」 |
| save_as | 保存ファイル名 | 空いたら登録者_画像URL追加 |
| duplicate_handling | 重複処理 | 統合処理をエラーにする / 統合処理をエラーにせず実行する |
| update_setting | 更新設定 | 更新する / 更新しない |
