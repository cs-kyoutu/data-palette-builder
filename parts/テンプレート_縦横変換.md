# テンプレート 縦持ち→横持ち変換

## フォーマット

```
『テンプレート『顧客ごとに縦持ちのデータを横に並べて変換』』
{aggregation_keys}を集約のキーとして選択し、[適用]を押下する
[横並びにしたいカラム]に{horizontal_columns}の順で選択し、[適用]を押下する
[並び順カラム]に「{sort_column}」《{sort_order}》を選択し、[適用]を押下する
[横並びにしたいカラムの数]を、カラムの並び順で設定した順番をもとに、上位《{top_n}》位まで横に並べるとし、[適用]を押下する
```

## 変数

| 変数 | 説明 | 例 |
|------|------|-----|
| aggregation_keys | 集約キー（「」区切り） | 「顧客ID」「メールアドレス」「姓」「名」 |
| horizontal_columns | 横並びカラム（「」区切り、順序重要） | 「商品ID」「品番」「商品名」「商品詳細ページURL」「商品画像URL」「現在価格」 |
| sort_column | ソートカラム | ランキング / 初回問合わせ日 |
| sort_order | ソート順 | 昇順 / 降順 |
| top_n | 上位何位まで | 5 / 8 / 10 |

## 後続のカラム名変更

横持ち変換後、b→dashは自動的にカラム名に(N)を付与する。リネームが必要：

```
『カラム名の変更』
「{original}」を""{renamed_1}""に変更する
「{original}(1)」を""{renamed_2}""に変更する
「{original}(2)」を""{renamed_3}""に変更する
...
```

命名規則: ①②③ or _1 _2 _3（アウトプット定義に合わせる）
